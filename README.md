## 🎯 Principles I follow

1. **Official sources only.** AWS docs, Microsoft Learn, kubernetes.io, Linux Foundation, OWASP, CNCF.
2. **Hands-on > theory.** If you can't build it, you don't know it.
3. **One cloud at first.** Master one, then sample others.
4. **Linux is the floor, not the ceiling.** Every cloud cert assumes you know it.
5. **Certifications are checkpoints, not destinations.** The job comes from projects + experience.
