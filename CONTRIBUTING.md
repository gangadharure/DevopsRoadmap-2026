# Contributing to Roadmap 2026

Thanks for wanting to help!

## What we accept

✅ Updated **official** links (vendor docs, foundation pages, free first-party courses)
✅ Corrections to exam details (cost, duration, passing score, validity)
✅ New roadmaps for foundational roles (Networking, Python, Go, etc.)
✅ Capstone project ideas (must be doable on free tier)
✅ Typos and grammar fixes

## What we don't accept

❌ Paid course promotion (KodeKloud, Udemy, A Cloud Guru, etc.)
❌ Personal blog / YouTube channel links
❌ Affiliate links of any kind
❌ Tools without their official documentation URL
❌ Long-form opinion pieces

## How to contribute

1. Fork the repo
2. Create a branch: `feat/<short-description>` or `fix/<short-description>`
3. Make your changes
4. Open a PR with a clear description of what changed and **why**
5. Reference an official source for any factual change

## Style

- Markdown headings start at `#` per file
- Use tables for comparisons
- Prefer **bold** for terms over `code` formatting unless it's literally code
- Keep emoji minimal in roadmap bodies (cover/README is fine)
- Cite official URLs inline as `[name](https://…)`

## Code of Conduct

Be kind. Be specific. Be useful.
